<Section id="introduction">

<h2 class="screen-reader-only">Introduction</h2>

<TextBlock>

Backdrops provide a simple way to place translucent media behind contextual content. Backdrops offer two style variants: <a href="#image-proportional">Image Proportional</a> and <a href="#fixed-height">Fixed Height</a>.

</TextBlock>

<AccessibilitySpotlight>

Backdrops remove the need for CSS background images. Meaningful images can use helpful *alt* text, and decorative images can use empty *alt* text.

Video backdrops respect users' reduced motion settings. This ensures a smoother, more comfortable experience for those who have opted for less motion on their devices.

</AccessibilitySpotlight>

</Section>

<TextBlock>

## Image Proportional

Image-proportional backdrops adjust automatically to different screen sizes. Below is an example of a basic Image Proportional backdrop.

</TextBlock>

<FigureExample
  figureNumber="1"
  highlightedLines="1,3-11,13-24"
  code={`
  <div class="backdrop theme-dark">

    <div class="backdrop__media">

      ${codeImgPlaceholderBackdrop}

      <div class="backdrop__media__credit">
        <p>Photo by <a href="#1">Jane Doe</a> on Unsplash</p>
      </div>

    </div>

    <div class="backdrop__cover">

      <div class="container text-align-center">
        <p class="h4 text-shadow margin-bottom-3">
          <em>In Their Eternal Faces<br>The Sun—with just delight</em>
        </p>
        <a class="button font-size-md theme-primary" href="#1">
          Emily Dickinson Poetry
        </a>
      </div>

    </div>

  </div>
  `}
>
  <ImageProportional />
</FigureExample>

<TableLineDescBlock targetFigure="1">

<TableLineDescRow targetFigure="1" lines="1">

The parent `.backdrop` class is required.

The `.theme-dark` class is optional. It sets the background and text color. If not present, these colors are inherited from the backdrop's nearest ancestor with a theme color applied.

</TableLineDescRow>

<TableLineDescRow targetFigure="1" lines="3-11">

The `.backdrop__media` class is required. It contains the feature image.

The `.backdrop__media__credit` class is optional. It provides a credit for the image.

</TableLineDescRow>

<TableLineDescRow targetFigure="1" lines="13-24">

The `.backdrop__cover` class is required. It holds the backdrop's contextual content.

</TableLineDescRow>

</TableLineDescBlock>

<TextBlock>

The size of this backdrop (proportional width and height) is determined by the size of its feature image.

The translucent effect of the backdrop is achieved with additional utility classes. Our <a href="/docs/color/#theme-color-utilities">theme color utilities</a> (e.g., `.theme-dark`) set the background and text color. The <a href="/docs/opacity/#opacity-utilities">opacity utility class</a> (e.g., `.opacity-40`) applied to the backdrop's feature image allows the background color to show through.

</TextBlock>

<TextBlock>

## Stack Breakpoint Modifiers

When using image size proportional backdrops for featured content (like the full-width example below), the backdrop may become too tall for the image on smaller viewport sizes.

</TextBlock>

<Stack />

<TextBlock>

To adjust for small screens, add a *stack breakpoint modifier* (e.g., `.backdrop--stack--{x}`) to the `.backdrop` class. Where ***x*** is the desired breakpoint (e.g., *lg* for large). The backdrop's image is placed above the content vertically until the viewport reaches the specified breakpoint.

Resize your browser window to see this in action. Notice the image is at full opacity because <a href="/docs/opacity/#opacity-utilities">opacity utilities</a> have their own breakpoint modifiers. Adding `.opacity-40--lg` to the image or its wrapper gives that image a specified opacity (40%) at the large (lg) breakpoint.

Figure 2 below provides the code used to create the backdrop above. Refer specifically to line 1, where the `.backdrop--stack--lg` breakpoint modifier is present.

</TextBlock>

<FigureExample
  figureNumber="2"
  highlightedLines="1"
  code={`
  <div class="backdrop backdrop--stack--lg theme-dark">

    <div class="backdrop__media">

      ${codeImgPlaceholderBackdrop}

      <div class="backdrop__media__credit">
        <p>Photo by <a href="#1">Jane Doe</a> on Unsplash</p>
      </div>

    </div>

    <div class="backdrop__cover">

      <div class="container narrow margin-y-5">

        <div class="text-shadow text-align-center margin-bottom-4">
          <h1 class="banner-headline">
            The mountain at a given distance<br>
            In amber lies
          </h1>
          <p>—Excerpt from <em>Delight becomes pictorial</em></p>
        </div>

        <div class="grid grid--column-2--md gap-2">

          <a class="button button--outline width-100" href="#1">
            About Emily Dickinson
          </a>

          <a class="button theme-secondary width-100 box-shadow-1" href="#1">
            Emily Dickinson Poetry
          </a>

        </div>

      </div>

    </div>

  </div>
  `}
/>

<TextBlock>

All of this together makes it easy to create responsive backdrops with different themes and content. What if we don't want our backdrop's size to be dictated by its feature image?

</TextBlock>

<TextBlock>

## Video Backdrop

The Backdrop component can also include a video as its feature media. By using the `.backdrop__media` class to contain a video element, you can create a dynamic and visually appealing backdrop that plays automatically when users access the page.

Below is an example of a Backdrop with a video as the feature media, including a media control button:

</TextBlock>

<Video />

<TextBlock>

To ensure the video respects users' preferences regarding reduced motion, you can include an optional control element using the `.backdrop__media__control` selector. This control allows users to pause the video if they prefer to limit motion effects. The button inside `.backdrop__media__control` must have the `data-check-reduced-motion` attribute, which automatically pauses the video for users who have enabled the "reduce motion" setting in their system preferences.

</TextBlock>

<FigureExample
  figureNumber="4"
  highlightedLines="5-14,16-25"
  code={`
  <div class="backdrop backdrop--stack--lg" id="introduction">

    <div class="backdrop__media">

      <div class="backdrop__media__control margin-y-2 container">
        <button
          class="button button--icon-only font-size-md border-radius-circle"
          aria-label="Pause video"
          aria-pressed="true"
          data-check-reduced-motion
          type="button">
          <span class="icon icon-pause" aria-hidden="true"></span>
        </button>
      </div>

      <video
        class="background-video opacity-30--lg gradient-mask-bottom"
        autoplay
        muted
        loop
        poster="path/to/your/video-poster.jpg"
        aria-hidden="true"
      >
        <source src="path/to/your/video.mp4" type="video/mp4">
      </video>

    </div>

    <div class="backdrop__cover align-content-end">

      <div class="container medium text-align-center margin-y-4">

        <h1 class="banner-headline text-shadow--lg margin-bottom-4 text-shadow">
          Faucibus scelerisque eleifend donec pretium vulputate
        </h1>

        <a class="button theme-primary box-shadow-1--lg" href="#1">
          <span class="text">Augue Eget Arcu</span>
          <span class="icon icon-arrow-right"></span>
        </a>

      </div>

    </div>

  </div>
  `}
/>

<TableLineDescBlock targetFigure="4">

<TableLineDescRow targetFigure="4" lines="5-14">

The `.backdrop__media__control` class is optional. It positions the video control.

The button has a label, a pressed state, and the `data-check-reduced-motion` attribute so Natura11y can pause the video when a user prefers reduced motion.

</TableLineDescRow>

<TableLineDescRow targetFigure="4" lines="16-25">

The `video` element is the backdrop media. It includes the `autoplay`, `muted`, and `loop` attributes so the video can play automatically by default.

</TableLineDescRow>

</TableLineDescBlock>
