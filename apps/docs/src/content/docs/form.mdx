<h2 className="screen-reader-only">
    Introduction
</h2>

<TextBlock>

Forms are notoriously challenging to style and make accessible. Natura11y provides basic style and behavior for building user friendly, accessible forms.

</TextBlock>

<AccessibilitySpotlight>

Form elements include a consistent visible focus for keyboard users.

Include accessible field labels <a href="https://www.w3.org/WAI/tutorials/forms/labels/#associating-labels-implicitly" target="_blank">implicitly</a> or <a href="https://www.w3.org/WAI/tutorials/forms/labels/#associating-labels-explicitly" target="_blank">explicitly</a>—it's up to you.

Intuitively mark *required* fields. When doing so, Natura11y adds *required* and *data-required* attributes automatically.

Checkbox and radio groups use semantic *fieldset* and *legend* tags.

Respects `prefers-reduced-motion` — the switch toggle transition is disabled when this preference is active.

When the search clear button is activated, Natura11y announces "Search cleared" to screen readers via an ARIA live region — no extra markup needed.

</AccessibilitySpotlight>

<TextBlock>

## Form Entries

The form entry parent selector (`.form-entry`) wraps each entry within a form. Child elements within form entries vary depending on their input types. Such types include *text inputs*, *textareas*, *selects*, *checkboxes*, and *radios*.

In Figure 1, the `.form-entry` selector contains an `<input>` field with the `type="text"` attribute.

</TextBlock>

<FigureExample
    figureNumber="1"
    lineAnnotations={[
        { range: '1' },
        { range: '3-18' },
        { range: '20-22' },
    ]}
    code={`
        <div class="form-entry">

            <label class="form-entry__field">

                <span class="form-entry__field__label">
                    Form Entry Label
                </span>

                <span class="form-entry__field__input">
                    <input
                        type="text"
                        name="textInputExample"
                        id="text-input-example"
                        aria-describedby="help-text-input-example"
                    />
                </span>

            </label>

            <small class="form-entry__help" id="help-text-input-example">
                Form entry help text
            </small>

        </div>
    `}
>
    <FormEntry />
</FigureExample>

<TableLineDescBlock targetFigure="1">

<TableLineDescRow targetFigure="1" lines="1">

The `.form-entry` parent selector is required.

It contains each entries elements, including help text and feedback messages.

When more than one `.form-entry` selector is present, spacing between them occurs automatically.

</TableLineDescRow>

<TableLineDescRow targetFigure="1" lines="3-18">

The `.form-entry__field` child selector is required.

It holds the field's label and input elements.

The `.form-entry__field__label` grandchild selector is required.

It holds the actual label text, and required indicator (when present).

The `.form-entry__field__input` grandchild selector is required.

It holds the field's actual input elements.

When using help text, `aria-describedby` is present on the `input` tag for assistive technology. It equals the help text's `id` attribute (below).

</TableLineDescRow>

<TableLineDescRow targetFigure="1" lines="20-22">

The `.form-entry__help` child selector is not required.

Its id equals its associated `aria-describedby` attribute.

</TableLineDescRow>

</TableLineDescBlock>

<TextBlock>

## Search Entry

Use the `.form-entry--search` modifier on the `.form-entry` element to create an accessible search field. A `[data-search-clear]` button inside `.form-entry__field__input` is shown automatically by Natura11y's JavaScript when the field has a value, and hidden when it is empty.

The label can be visually hidden with `.screen-reader-only` — a visible label is not required when a leading search icon makes the purpose clear.

</TextBlock>

<FigureExample
    figureNumber="2"
    lineAnnotations={[
        { range: '1' },
        { range: '10' },
        { range: '16-23' },
        { range: '25-27' },
    ]}
    code={`
        <div class="form-entry form-entry--search">

            <label class="form-entry__field">

                <span class="form-entry__field__label screen-reader-only">
                    Search
                </span>

                <span class="form-entry__field__input">
                    <span class="icon icon-search opacity-50" aria-hidden="true"></span>
                    <input
                        type="search"
                        id="search-example"
                        name="searchExample"
                    />

                    <button
                        type="button"
                        class="button button--icon-only"
                        data-search-clear
                        aria-label="Clear search"
                        aria-controls="search-example">
                            <span class="icon icon-clear" aria-hidden="true"></span>
                    </button>

                    <button type="submit" class="button theme-primary">
                        Search
                    </button>
                </span>

            </label>

        </div>
    `}
>
    <FormEntrySearch />
</FigureExample>

<TableLineDescBlock targetFigure="2">

<TableLineDescRow targetFigure="2" lines="1">

The `.form-entry--search` modifier on `.form-entry` is required.

It suppresses the browser's native clear button on `<input type="search">` so the custom `[data-search-clear]` button is used instead.

</TableLineDescRow>

<TableLineDescRow targetFigure="2" lines="10">

The leading icon is optional. When present, the input's `padding-inline-start` is removed automatically so the icon and text align cleanly.

</TableLineDescRow>

<TableLineDescRow targetFigure="2" lines="16-23">

The `[data-search-clear]` button is required. Natura11y's JavaScript hides it by default and shows it only when the field has a value.

The `aria-controls` attribute must equal the `id` of the associated `<input>`.

</TableLineDescRow>

<TableLineDescRow targetFigure="2" lines="25-27">

The submit button is optional. Use `type="submit"` when the search field is inside a `<form>` element.

</TableLineDescRow>

</TableLineDescBlock>

<TextBlock>

### Pre-populated Search (Search Results Page)

When linking to a search results page, pre-apply the `has-value` class to the `.form-entry--search` element and set the `value` attribute on the input to reflect the active query. This ensures the clear button is visible on page load — no JavaScript required for that initial state. It's a small detail that makes a meaningful difference in usability.

The <LinkOpenNew LinkUrl="https://cavidano.github.io/natura11y/dist/html/templates/search-results/" LinkText="Search Results Template" /> demonstrates this pattern in a full page layout, including a refined search field, filter sidebar, results list, and pagination.

</TextBlock>

### Single Checkbox

For a single checkbox, do not use a `<fieldset>` and `<legend>`. Instead, provide an id on the `.form-entry__field__label` selector (Figure 7, line 5). Then associate it with `aria-labelledby` placed on the surrounding `<label>` tag (Figure 7, line 13).

</TextBlock>

<FigureExample
    figureNumber="7"
    lineAnnotations={[
        { range: '5' },
        { range: '13' },
    ]}
    code={`
        <div class="form-entry">
            <div class="form-entry__field">
                <span class="form-entry__field__label" id="single-option-label">
                    Form Entry Label
                </span>
                <div class="form-entry__option">
                    <div class="form-entry__option__check">
                        <label aria-labelledby="single-option-label">
                            <input
                                type="checkbox"
                                name="singleOption"
                                id="single-option"
                                value="option">
                                <span class="option__label">
                                    Option
                                </span>
                        </label>
                    </div>
                </div>
            </div>
        </div>
    `}
>

</FigureExample>

<TextBlock>

## Required Fields

Natura11y makes it easy to mark required fields. Add `data-required="true"` to the parent `.form-entry` element of the required field. On the same `.form-entry` element, include a custom error message using the `data-error-message={x}` attribute. Here <strong><em>x</em></strong> equals the message displayed to the user during the validation process.

</TextBlock>

<FigureExample
    figureNumber="10"
    lineAnnotations={[
        { range: '3-4' },
    ]}
    code={`
        <div
            class="form-entry"
            data-required="true"
            data-error-message="Custom error message">
            <div class="form-entry__field">
                ...
            </div>
        </div>
    `}
>
    <RequiredFields />
</FigureExample>

<TextBlock>

The two data attributes mentioned above are shown in Figure 10, lines 3 and 4. When present on the `.form-entry` selector, Naturally's Javascript adds the necessary `required` and `data-required` attributes to the form entry's child `&lt;input&gt;` element automatically.

With the previous example, notice the red asterisk placed *before* the label. This too, is added automatically to required fields (via Natura11y's CSS file). The asterisk has become the standard identifier for required fields.

</TextBlock>

## Form Validation

Natura11y provides superficial client-side form validation. Include the `novalidate` attribute on the `<form>` tag.

</TextBlock>

<FigureExample
    figureNumber="11"
    code={`

            <div
                class="form-entry"
                data-required="true"
                data-error-message="Username is required!">

                <label class="form-entry__field form-entry__field--float">
                    <span class="form-entry__field__label">
                        Username
                    </span>
                    <span class="form-entry__field__input">
                        <input type="text" id="account-username" name="accountUsername">
                    </span>
                </label>

            </div>

        </form>
    `}
>
