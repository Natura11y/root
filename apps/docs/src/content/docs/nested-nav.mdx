<h2 className="screen-reader-only">
    Introduction
</h2>

<TextBlock>

The nested navigation component provides a structured, multi-level sidebar navigation. It supports up to three levels of hierarchy — with indentation and font size communicating depth, and a visual indicator bar marking the active L1 item.

No JavaScript is required. The component is pure CSS.

</TextBlock>

<AccessibilitySpotlight>

Wrap the nested nav in a `<nav>` element with a descriptive `aria-label` to identify it as a navigation landmark.

Add `aria-current="page"` to the link that represents the current page. The active indicator bar responds to any `[aria-current]` value, so `aria-current="true"` can mark a parent section as active when a child page is selected.

</AccessibilitySpotlight>

</Section>

<TextBlock>

## Nested Navigation

Add `.nested-nav` to a `<ul>` element (Figure 1).

</TextBlock>

<FigureExample
    figureNumber="1"
    highlightedLines="1,2,7"
    code={`<nav aria-label="Field Guide Navigation">
  <ul class="nested-nav">
    <li>
      <a href="#">Overview</a>
    </li>
    <li>
      <a href="#" aria-current="page">American Robin</a>
      <ul>
        <li>
          <a href="#">Nesting</a>
        </li>
        <li>
          <a href="#">Diet</a>
        </li>
        <li>
          <a href="#">Migration</a>
        </li>
      </ul>
    </li>
    <li>
      <a href="#">Conservation</a>
    </li>
  </ul>
</nav>`}
>
    <Default />
</FigureExample>

<TableLineDescBlock targetFigure="1">

<TableLineDescRow targetFigure="1" lines="1">

The `<nav>` element with `aria-label` identifies this as a navigation landmark for assistive technology. Use a label that distinguishes it from other navigation regions on the page, such as `"Section Navigation"`.

</TableLineDescRow>

<TableLineDescRow targetFigure="1" lines="2">

Add `.nested-nav` to any `<ul>` element to apply the nested navigation styles.

</TableLineDescRow>

<TableLineDescRow targetFigure="1" lines="7">

Add `aria-current="page"` to the link representing the current page. This drives the active indicator bar at level 1, and semantically communicates location to assistive technology at any level.

</TableLineDescRow>

</TableLineDescBlock>

</Section>

<Section id="active-states">

<TextBlock>

## Active States

When a sub-item is the current page, mark the L1 parent with `aria-current="true"` to activate the indicator bar on the section — and mark the active sub-item with `aria-current="page"` (Figure 2).

</TextBlock>

<FigureExample
    figureNumber="2"
    code={`<ul class="nested-nav">
  <li>
    <a href="#">Overview</a>
  </li>
  <li>
    <a href="#" aria-current="true">American Robin</a>
    <ul>
      <li>
        <a href="#" aria-current="page">Nesting</a>
      </li>
      <li>
        <a href="#">Diet</a>
      </li>
      <li>
        <a href="#">Migration</a>
      </li>
    </ul>
  </li>
  <li>
    <a href="#">Conservation</a>
  </li>
</ul>`}
>
    <ActiveStates />
</FigureExample>

<TextBlock>

The active indicator bar only appears on level-1 items. It responds to any `[aria-current]` value on the L1 link, and to `[aria-selected="true"]` — giving you flexibility in how you communicate hierarchy to assistive technology.

</TextBlock>

</Section>
