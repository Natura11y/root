<h2 className="screen-reader-only">
    Introduction
</h2>

<TextBlock>

The main menu is a complete responsive header system with four preassigned areas: ***logo***, ***nav***, ***search***, and ***actions***. Two layout variants are available: ***bar*** places the menu *inline* with the logo, while ***stack*** places it *below*, spanning the full width. Both variants share the same child selectors and differ only in their parent modifier class.

</TextBlock>

<AccessibilitySpotlight>

The main menu uses `role="navigation"` via the `<nav>` element. Include `aria-label` to distinguish it from other navigation landmarks on the page.

Mobile toggle buttons use `aria-controls` to reference the menu or search `id`, and `aria-expanded` to communicate open and closed state to assistive technology.

Learn more about <a href="https://www.w3.org/TR/WCAG21/#content-on-hover-or-focus" target="_blank">visible hover and focus</a>.

</AccessibilitySpotlight>

</Section>

<TextBlock>

## Main Menu Bar

The `.main-menu--bar--{bp}` selector places the main menu *inline* with the logo. Here, ***bp*** is the responsive breakpoint at which the component transitions from its mobile stacked layout to the desktop inline layout (e.g. *lg* for large). At this breakpoint the toggle buttons are hidden and the full menu is displayed.

</TextBlock>

<TableLineDescBlock targetFigure="1">

<TableLineDescRow targetFigure="1" lines="1">

The `.main-menu--bar--{bp}` block-level selector is required.

It accepts two modifiers: the variant (*bar* or *stack*) and the responsive breakpoint. At the breakpoint, the component transitions from its mobile layout to the desktop inline layout.

</TableLineDescRow>

<TableLineDescRow targetFigure="1" lines="2-10">

The `.main-menu__logo` selector is required.

It contains the website's logo and home link.

When using a single-color `<svg>` for the logo, the `data-logo="brand"` attribute placed on the `<a>` tag sets the SVG fill color to `currentColor`. This is useful when a theme color utility is present on a parent element.

</TableLineDescRow>

<TableLineDescRow targetFigure="1" lines="12-44">

The `.main-menu__nav` selector is required.

The `id` attribute must match the `aria-controls` value of the associated mobile toggle button found within `.main-menu__toggle`.

Include `aria-label` on the `<nav>` tag to provide a descriptive name for assistive technology.

Learn about <a href="/docs/dropdown/">dropdowns</a>.

</TableLineDescRow>

<TableLineDescRow targetFigure="1" lines="46-70">

The `.main-menu__toggle` selector is required.

It contains the mobile toggle buttons for the menu and search. Both are visible on mobile and hidden at the component's breakpoint.

The search toggle button is only needed in the Main Menu Bar. In the Main Menu Stack, the search field remains visible from mobile to desktop.

Each button uses `data-toggle="collapse"` and `data-target-close` to mutually dismiss the other panel when opened.

Learn about <a href="/docs/collapse/">collapse</a>.

</TableLineDescRow>

<TableLineDescRow targetFigure="1" lines="72-85">

The `.main-menu__search` selector is optional.

The `role="search"` attribute on the `<form>` tag is present for assistive technology.

The `data-focus-first` attribute moves focus to the search input when the field is revealed.

Learn about <a href="/docs/form/#form-entries">form entries</a>.

</TableLineDescRow>

<TableLineDescRow targetFigure="1" lines="87-96">

The `.main-menu__actions` selector is optional.

It contains action buttons that should persist on every page (e.g. *Sign In*, *Translate*).

</TableLineDescRow>

</TableLineDescBlock>

<TextBlock>

## Main Menu Stack

The `.main-menu--stack--{bp}` selector places the main menu *below* the logo row, spanning the full width of the component. On desktop the logo row displays alongside the search and actions areas. The menu appears beneath on its own full-width row.

</TextBlock>

<TextBlock>

Both variants use the same HTML markup and child selectors. The only differences in the stack variant are:

1. The parent class is `.main-menu--stack--{bp}` instead of `.main-menu--bar--{bp}`
2. The `<nav>` element is placed *last* in source order, after the toggle, search, and actions areas
3. The search toggle button inside `.main-menu__toggle` is *not* required — the search field remains visible from mobile to desktop

</TextBlock>

</TableLineDescRow>

<TableLineDescRow targetFigure="2" lines="29-31">

In the stack variant, the `.main-menu__nav` `<nav>` is placed *last* in source order.

The CSS grid layout positions it below the logo row regardless of source order, but placing it last ensures a logical reading order for assistive technology.
