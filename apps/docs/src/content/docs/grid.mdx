<TextBlock>

## CSS Grid

Natura11y's CSS Grid component offers essential layout options through utility classes.

</TextBlock>

<GridOverview />

<TextBlock>

CSS grid layouts have limitless possibilities, and Natura11y only covers the essentials. You can and *should* write custom CSS grid layouts alongside Natura11y's styles.

Let's look at the `.grid` component beginning with Figure 1.

</TextBlock>

<FigureExample
    figureNumber="1"
    lineAnnotations={[
        { range: '1' },
        { range: '3-13' },
    ]}
    code={`
        <div class="grid">

            <div>
                <p>01</p>
            </div>

            <div>
                <p>02</p>
            </div>

            <div>
                <p>03</p>
            </div>

        </div>
    `}
>
    <GridStack />
</FigureExample>

<TableLineDescBlock targetFigure="1">

<TableLineDescRow targetFigure="1" lines="1">

The parent `.grid` selector is required.

It applies the css grid specification.

</TableLineDescRow>

<TableLineDescRow targetFigure="1" lines="3-13">

All direct children of the `.grid` selector are automatically grid items.

</TableLineDescRow>

</TableLineDescBlock>

<TextBlock>

### Grid Columns

To create columns within a `.grid` selector, use the `.grid--column-{x}` modifier, where ***x*** represents the number of columns (from 1 to 12).

For example, in Figure 2, the `.grid--column-3` class creates a three-column grid.

</TextBlock>

<FigureExample
    figureNumber="2"
    highlightedLines="1"
    code={`
        <div class="grid grid--column-3">
            ...
        </div>
    `}
>
    <GridColumnWidth />
</FigureExample>

<TextBlock>

### Column Span

Grid items can span multiple columns within their parent `.grid--column-{x}` selector. Use the `.grid__colspan-{x}` modifier to specify the number of columns the item should span (from 2 to 12).

</TextBlock>

<FigureExample
    figureNumber="3"
    highlightedLines="3"
    code={`
        <div class="grid grid--column-4">

            <div class="grid__colspan-3">
                <p>01</p>
            </div>

            ...

        </div>
    `}
>
    <GridColumnSpan />
</FigureExample>

<TextBlock>

### Grid Rows

Rows in the grid component are automatically created. Grid items can span up to 6 rows by using the `.grid__rowspan-{x}` modifier, where ***x*** represents the number of rows (from 2 to 6). See Figure 4, line 3.

</TextBlock>

<FigureExample
    figureNumber="4"
    highlightedLines="3"
    code={`
        <div class="grid grid--column-2">

            <div class="grid__rowspan-2">
                <p>01</p>
            </div>

            ...

        </div>
    `}
>
    <GridRowSpan />
</FigureExample>

<TextBlock>

### Breakpoint Modifiers

All column and row utilities in `.grid` support breakpoint modifiers.

For responsive layouts, add a breakpoint modifier (e.g., `.grid--column-{x}--{y}`) where ***x*** is the column count and ***y*** is the desired breakpoint.

For instance, to create a three-column layout at the medium breakpoint, use `.grid--column-3--md` (Figure 5, line 1).

</TextBlock>

<FigureExample
    figureNumber="5"
    highlightedLines="1"
    code={`
        <div class="grid grid--column-3--md">
            ...
        </div>
    `}
>
    <GridColumnWidthBreakpoints />
</FigureExample>

<TextBlock>

### Grid Divider Modifier

You can add horizontal and vertical borders between grid cells by applying the `.grid--divider` modifier class to `.grid` selectors.

</TextBlock>

<FigureExample
    figureNumber="6"
    highlightedLines="1"
    code={`
        <div class="grid grid--divider grid--column-3">
            ...
        </div>
    `}
>
    <GridDividers />
</FigureExample>

<TextBlock>

You can specify which axis (*x* or *y*) has continuous lines. Chain `--fill-x` or `--fill-y` modifiers to `.grid--divider`. With `.grid--divider--fill-x` present on the example in Figure 7, the horizontal lines run through.

</TextBlock>

<FigureExample
    figureNumber="7"
    highlightedLines="1"
    code={`
        <div class="grid grid--divider--fill-x grid--column-3">
            ...
        </div>
    `}
>
    <GridDividers gridDividerClass="grid--divider--fill-x" />
</FigureExample>

<TextBlock>

With `.grid--divider--fill-y` present on the example in Figure 8, the vertical lines run through.

</TextBlock>

<FigureExample
    figureNumber="8"
    highlightedLines="1"
    code={`
        <div class="grid grid--divider--fill-y grid--column-3">
            ...
        </div>
    `}
>
    <GridDividers gridDividerClass="grid--divider--fill-y" />
</FigureExample>

<TextBlock>

The `.grid--divider` modifier class adds borders using pseudo-elements on each of its direct children or *cells*. For this reason, when its child has `overflow: hidden;` CSS property applied, such as with <a href="/docs/backdrop/">backdrops</a>, for example, that child's border would not appear.

To remedy this, simply nest each child of `.grid--divider` in a containing element. You could use a `<ul>` tag with `<li>` children, as shown in Figure 9:

</TextBlock>

<FigureExample
    figureNumber="9"
    code={`
        <ul class="grid grid--divider--fill-x grid--column-3">

            <li>
                <div class="backdrop">
                    <!-- ... -->
                </div>
            </li>

            <li>
                <div class="backdrop">
                <!-- ... -->
                </div>
            </li>

            <li>
                <div class="backdrop">
                <!-- ... -->
                </div>
            </li>

        </ul>
    `}
/>

</Section>

## Order Utilities

Change the layout <a href="https://developer.mozilla.org/en-US/docs/Web/CSS/order" target="_blank">order</a> of grid items.

</TextBlock>

<FigureExample
    figureNumber="10"
    code={`
        <div class="grid grid--column-3--md theme-light border font-size-md text-align-center">

            <div class="padding-2 border order-3">
                <p>01</p>
            </div>
            <div class="padding-2 border">
                <p>02</p>
            </div>
            <div class="padding-2 border">
                <p>03</p>
            </div>

        </div>

        <!-- All order utilities: -->

        <div class="order-1">...</div>
        <div class="order-2">...</div>
        <div class="order-3">...</div>
        <div class="order-4">...</div>
        <div class="order-5">...</div>
        <div class="order-6">...</div>
        <div class="order-7">...</div>
        <div class="order-8">...</div>
        <div class="order-9">...</div>
        <div class="order-10">...</div>
        <div class="order-11">...</div>
        <div class="order-12">...</div>
    `}
>

</FigureExample>

<TextBlock>

## Sidebar Layouts

The sidebar layout components provide fixed-width sidebar columns. Columns stack vertically on mobile and shift into a side-by-side layout at the responsive breakpoint. The sidebar width is controlled by the `--grid-sidebar-width` CSS custom property (default: `300px`).

### Left Sidebar

Use `.grid-sidebar--left` to place a fixed-width sidebar on the left with the main content occupying the remaining space.

</TextBlock>

<div class="container wide">
    <GridSidebarLeft />
</div>

<FigureExample
    figureNumber="12"
    lineAnnotations={[
        { range: '1' },
        { range: '3' },
        { range: '7' },
    ]}
    code={`
        <div class="grid-sidebar--left">

            <div class="grid-sidebar__minor">
                ...
            </div>

            <div class="grid-sidebar__major">
                ...
            </div>

        </div>
    `}
/>

<TableLineDescBlock targetFigure="12">

<TableLineDescRow targetFigure="12" lines="1">

`.grid-sidebar--left` positions the sidebar on the left at the `lg` breakpoint. On smaller viewports, the sidebar stacks above the main content.

</TableLineDescRow>

<TableLineDescRow targetFigure="12" lines="3">

`.grid-sidebar__minor` assigns this element to the sidebar column.

</TableLineDescRow>

<TableLineDescRow targetFigure="12" lines="7">

`.grid-sidebar__major` assigns this element to the main content column.

</TableLineDescRow>

</TableLineDescBlock>

<TextBlock>

### Right Sidebar

Use `.grid-sidebar--right` to place the sidebar on the right side instead.

</TextBlock>

    <GridSidebarRight />

<TableLineDescBlock targetFigure="13">

<TableLineDescRow targetFigure="13" lines="1">

</TableLineDescRow>

<TableLineDescRow targetFigure="13" lines="3">

`.grid-sidebar__minor` assigns this element to the sidebar column.

</TableLineDescRow>

<TableLineDescRow targetFigure="13" lines="7">

`.grid-sidebar__major` assigns this element to the main content column.

</TableLineDescRow>

</TableLineDescBlock>

<TextBlock>

### Sidebars on both sides

For a three-column layout with sidebars on both sides, use `.grid-sidebars`. The layout switches to three columns at the extra-large (`xl`) breakpoint.

</TextBlock>

<div class="container wide">
    <GridSidebars />
</div>

<FigureExample
    figureNumber="14"
    lineAnnotations={[
        { range: '1' },
        { range: '3' },
        { range: '7' },
        { range: '11' },
    ]}
    code={`
        <div class="grid-sidebars">

            <div class="grid-sidebar__minor-start">
                ...
            </div>

            <div class="grid-sidebar__major">
                ...
            </div>

            <div class="grid-sidebar__minor-end">
                ...
            </div>

        </div>
    `}
/>

<TableLineDescBlock targetFigure="14">

<TableLineDescRow targetFigure="14" lines="1">

`.grid-sidebars` creates a three-column layout at the `xl` breakpoint. On smaller viewports, all three areas stack vertically.

</TableLineDescRow>

<TableLineDescRow targetFigure="14" lines="3">

`.grid-sidebar__minor-start` assigns this element to the left sidebar column.

</TableLineDescRow>

<TableLineDescRow targetFigure="14" lines="7">

`.grid-sidebar__major` assigns this element to the main content column.

</TableLineDescRow>

<TableLineDescRow targetFigure="14" lines="11">

`.grid-sidebar__minor-end` assigns this element to the right sidebar column.

</TableLineDescRow>

</TableLineDescBlock>

<TextBlock>

Sidebar layouts are compatible with <a href="#css-grid">grid divider modifiers</a> for column borders and <a href="#gap-utilities">gap utilities</a> for spacing between areas.

</TextBlock>
